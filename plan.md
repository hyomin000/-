## 1.2 기능 구성도 (Functional Specification)
###  입력 및 분석 (F-01, F-02)
*   **문자열 수집**: 사용자로부터 의심스러운 메시지 입력 수신
*   **패턴 매칭**: DB에 저장된 피싱 키워드(대출, 검찰, 결제 등) 비교 분석
*   **위험도 스코어링**: 가중치 기반의 3단계(안전/주의/위험) 등급 산출

###  시각화 및 접근성 (F-03, F-13)
*   **결과 대시보드**: 등급별 배경색 변경(초록/노랑/빨강) 및 상태 메시지 출력
*   **고대비 UI**: 노약자를 위한 텍스트 크기 확대 및 직관적 아이콘 배치

###  관리 및 로그 (F-04, F-08, F-11)
*   **키워드 관리**: 최신 피싱 트렌드 반영을 위한 위험 단어 추가/삭제
*   **알림 로그**: 탐지 이력 저장 및 보호자 전송용 가상 메시지 생성

###  IoT 확장 기능 (F-05, F-06, F-07)
*   **시리얼 인터페이스**: 분석 데이터를 아두이노 패킷으로 변환 전송
*   **멀티모달 경고**: LED(시각) 및 부저(청각)를 활용한 2차 경보

## 1.4 시스템 모듈 및 데이터 명세

###  하드웨어 및 통신 연동 (Communication Bridge)

#### **1. Serial Manager (통신 브릿지)**
*   **역할**: 분석 결과 데이터를 시리얼 통신을 통해 Arduino로 전송
*   **안정성**: 통신 실패 시 예외 처리 및 재시도 로직 수행 (**F-05, F-12**)

#### **2. Hardware (Physical Feedback)**
*   **동작**: 위험 등급에 따른 LED 색상 제어 및 부저 출력 (**F-06, F-07**)
*   **등급별 신호 체계**:
    *   **안전(0)**: 초록 LED 점등
    *   **주의(1)**: 노랑 LED 점등
    *   **위험(2)**: 빨강 LED 점등 + 부저(Buzzer) 작동

## 3.4 시스템 구조 요약 (System Overview)

본 시스템은 **Java 기반 GUI**와 **Arduino 하드웨어**가 유기적으로 연동되는 피싱 탐지 솔루션입니다.

1.  **입력 및 분석**: 사용자가 문장을 입력하면 Java 엔진이 키워드를 탐지하고 위험 점수를 계산합니다.
2.  **다중 출력**: 분석 결과는 GUI 화면에 시각화됨과 동시에 시리얼 통신을 통해 Arduino로 전송됩니다.
3.  **물리적 피드백**: Arduino는 수신된 등급에 따라 LED 색상과 부저를 제어하여 직관적인 위험 알림을 제공합니다.
4.  **데이터 관리**: 모든 분석 결과 및 보호자 알림 로그는 데이터베이스에 저장되어 추후 조회가 가능합니다.
